3. **Product Name**: The name or description of the product. This would typically be stored as a `VARCHAR` data type.
4. **Cost**: The cost associated with producing or acquiring the product. You can use a `FLOAT` or `DECIMAL` data type to represent costs.
5. **Price**: The selling price of the product. You can use a `FLOAT` or `DECIMAL` data type to represent prices.
6. **Profit Margin**: The profit margin calculated as (Price - Cost) / Price. This would typically be stored as a `FLOAT` or `DECIMAL` data type.
7. **Date**: The date when the margin data was recorded. You can use a `DATE` data type for this column.
8. **Region/Store ID**: An identifier for the region or store associated with the margin data. This could be stored as an `INTEGER` or `VARCHAR` data type.
9. **Last Updated**: The date and time when the margin data was last updated. You can use a `DATETIME` or `TIMESTAMP` data type for this.
